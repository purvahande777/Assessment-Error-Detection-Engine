## **Deliverable 1: Error Detection Rules**

The following strict rules have been implemented to ensure fairness, system stability, and accurate evaluation:

### **Category A: Logical Inconsistencies**
1. **Total Question Count:** The system must serve exactly 10 questions per candidate session.
2. **Introduction Verification:** Question 1 must strictly be classified as the "Introduction" question.
3. **Time Limit Constraint:** A candidate cannot exceed the maximum time limit of 30 minutes (1800 seconds) per question.
4. **Negative Marking Accuracy:** Every incorrect answer must incur a precise penalty of -0.25 points.
5. **Difficulty Progression:** Question difficulty must follow a structured, gradual increase without erratic jumps.

### **Category B: System-Level & Security Errors**
1. **Face Detection Continuity:** The camera API must continuously return valid face detection data. `null` or timeouts will trigger a System Error.
2. **Multiple Person Detection:** If `faces_detected > 1`, a Security Alert is triggered immediately to prevent cheating.
3. **LLM Evaluation Validation:** The GPT/LLM evaluation response must be a valid numerical format. String responses or timeouts trigger a System Error.
